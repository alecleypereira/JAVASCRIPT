Introdução ao JavaScript

JavaScript

JavaScript foi criado para ser uma linguagem de programação focada no desenvolvimento web. Com ela, é possível criar interatividade, animações, validação de formulários, entre outras funcionalidades de forma fácil.

Criação do JavaScript

JavaScript foi criado por Brendan Eich em 1995 enquanto ele trabalhava na Netscape Communications Corporation. Originalmente, foi desenvolvido em apenas 10 dias e foi chamado de "Mocha", depois "LiveScript" e, finalmente, "JavaScript". A intenção era criar uma linguagem de script que pudesse ser usada tanto por programadores experientes quanto por iniciantes para desenvolver páginas web interativas.

Envolvimento com o ECMAScript

Em 1996, para evitar uma guerra de padrões e garantir a interoperabilidade entre diferentes navegadores, a Netscape submeteu JavaScript à Ecma International (European Computer Manufacturers Association). A linguagem foi padronizada sob o nome ECMAScript. O ECMAScript é a especificação oficial da linguagem, e JavaScript é a implementação mais popular dessa especificação.

Evolução do ECMAScript

Desde a sua padronização inicial, o ECMAScript passou por várias revisões importantes:

  • ECMAScript 1 (1997):Primeira edição.
  • ECMAScript 3 (1999): Amplamente adotada e usada por vários anos.
  • ECMAScript 5 (2009): Introduziu várias melhorias e novas funcionalidades.
  • ECMAScript 6 (2015): Também conhecido como ECMAScript 2015 ou ES6, trouxe uma grande quantidade de novas funcionalidades e melhorias, marcando uma evolução significativa da linguagem.
  • Atualizações anuais: Desde ES6, a especificação do ECMAScript é atualizada anualmente com novas funcionalidades e melhorias incrementais.

Mozilla e o Firefox

Mozilla tem sido uma força motriz na promoção e desenvolvimento de JavaScript através do seu navegador Firefox. O motor JavaScript do Firefox, conhecido como SpiderMonkey, foi o primeiro motor JavaScript, criado originalmente por Brendan Eich na Netscape. A Mozilla continua a contribuir significativamente para o avanço da linguagem JavaScript e da especificação ECMAScript através de suas iniciativas e participação em grupos de padronização.

Comitê TC39

  • TC39: O Comitê Técnico 39 da Ecma International é responsável pela padronização da linguagem ECMAScript (JavaScript). O TC39 é composto por membros de várias empresas e organizações, incluindo Mozilla, Google, Microsoft, Apple, entre outras.

Node.Js

Node.js é um ambiente de execução JavaScript no lado do servidor criado por Ryan Dahl em 2009. Ele usa o motor V8 do Google e é conhecido por sua arquitetura event-driven e operações de I/O não bloqueantes, tornando-o ideal para aplicações que exigem alta performance em operações I/O intensivas.

Aplicações Comuns

  • Servidores Web: Devido à sua eficiência em I/O, é muito usado para construir servidores web.
  • APIs em Tempo Real: Ideal para aplicações que requerem comunicação em tempo real, como chats e jogos online.
  • Ferramentas de Desenvolvimento: Amplamente usado para construir ferramentas e utilitários de desenvolvimento.

Variáveis

Variáveis são um conceito que normalmente se tem dificuldade de aprender no início, porém só é necessário entender como elas funcionam. Uma variável pode ser interpretada como uma caixa, e nessa caixa você pode guardar coisas; essas coisas podem ser de tudo: uma coordenada, um valor, um texto, etc. No JS, cada coisa que você guarda nessa caixa tem um tipo. Tipos de Declaração em JavaScript

`var`

  • Escopo: Funciona dentro do contexto de uma função ou globalmente.
  • Re-declarável: Pode ser redeclarada e redefinida.
  • Uso: Menos comum em código moderno devido a problemas com escopo.

`let`

  • Escopo: Funciona dentro do bloco onde é declarada (bloco, função, global).
  • Re-declarável: Não pode ser redeclarada no mesmo escopo.
  • Uso: Recomendada para variáveis que irão mudar de valor.

`const`

  • Escopo: Igual ao `let` (bloco, função, global).
  • Constante: Não pode ser redefinida após a atribuição inicial.
  • Uso: Recomendada para valores que não devem mudar.

OBS: Escopo é onde podemos chamar a variável.

Os nomes dados as variáveis são chamados de identificadores e eles tem regras para a sua nomenclatura:

  • Pode começar com letra, “$”, ou “_”.
  • Não podem começar com números.
  • É possível usar letras ou números.
  • É possível usar símbolos e acentos.
  • Não podem conter espaço.
  • Não podem ser palavras reservadas.

OBS: Palavras reservadas são palavras que o próprio JS usa em seus comandos.
